### О проекте

Платформа для проведения турниров по пляжному волейболу в 8 форматах.
Работают **3 агента параллельно**, координация через `STATUS.md`.

**Стек:** Vanilla JS (ES-модули), HTML/CSS, Node.js + Express (сервер), PostgreSQL (позже), Playwright + Vitest (тесты).

**Сервер:** VPS adminvps.ru, деплой через git push.

**Приоритет форматов:** IPT (уже есть) → Thai Mixed → King of the Court → Monster → Camp → ...

### Текущее состояние кода

Проект находится в папке `ФИНАЛ/`. Это рабочая сборка — **хаб уже работает**:

```
index.html              ← Хаб (ростер, рейтинг, история)
assets/
  app.css
  js/
    main.js             ← Bootstrap, глобальные loadHistory/saveHistory
    screens/
      core.js           ← Навигация, навбар, кнопка ✕ Выход
      home.js           ← Экран с турнирами и календарём
      roster.js         ← Ростер игроков, настройки турнира
      ipt.js            ← IPT-формат (работает внутри хаба)
      players.js        ← Профили игроков
      stats.js, svod.js, courts.js
    domain/
      players.js        ← Логика игроков
      timers.js         ← Таймеры
      tournaments.js    ← Данные турниров
    ui/
      ipt-format.js     ← UI для IPT (счёт, корты, таблица)
      roster-auth.js    ← Пароль ростера
      stats-recalc.js   ← Пересчёт статистики
      ...
ipt-session.html        ← IPT в отдельной вкладке (новый подход)
```

**Что уже сделано (не трогать без необходимости):**
- Хаб работает: хоум, ростер, IPT, игроки, статистика
- `loadHistory()` / `saveHistory()` — глобальные хелперы в `index.html`
- Кнопка **✕ Выход** возвращает на основной сайт (`SITE_URL` в `config.js`)
- IPT быстрый старт открывается в новой вкладке (`ipt-session.html`)
- `t.level||''` guard в `home.js` (защита от undefined)
- `noscript` и `sr-only` labels в `index.html` (accessibility)

**Чего ещё НЕТ:**
- папки `shared/` — общие модули (создаёт ARCH)
- папки `formats/` — форматы (Thai, KOTC и др.)
- папки `server/` — серверная часть API
- тесты Thai-формата

### Твои задачи — Этап 0

Прочитай `PLATFORM_ROADMAP.md` и `STATUS.md` полностью.

Затем выполняй задачи **только своей роли**:

**🔵 ARCH — начинай с A0.1:**
Создать `shared/` модули: `ui-kit.js`, `players.js`, `timer.js`, `table.js`, `utils.js`, `auth.js`, `base.css`.
Источники: извлечь из существующего кода (`ipt.js`, `ipt-format.js`, `domain/players.js`, `domain/timers.js`, `core.js`).
Правило: каждый модуль — чистые функции, без DOM-зависимостей между модулями.

**🟣 FORMAT — начинай с F0.1:**
Создать `formats/thai/thai-format.js` — чистые математические функции.
НОЛЬ зависимостей от хаба. Можно начинать сразу, не ждать ARCH.
Контракт функций описан в `PLATFORM_ROADMAP.md` → раздел F0.1.

**🟢 QA — начинай с Q0.1:**
Настроить тестовую инфраструктуру: vitest для unit-тестов, Playwright для smoke.
Написать заготовки тестов для `thai-format.js` по контракту из роадмапа — **параллельно с FORMAT**.

### Правила работы

```
ПЕРЕД задачей:
1. Прочитать STATUS.md
2. Проверить: мои зависимости отмечены DONE?
3. Если нет — взять другую задачу или написать в BLOCKED

ПОСЛЕ задачи:
1. npm run test:unit (если есть тесты)
2. Обновить STATUS.md:
   - [x] Задача ✅ (дата)
   - Файлы: список
   - Разблокирует: список зависимых задач
3. Если сломал чужой тест → СТОП → записать в BLOCKED

НЕ ТРОГАТЬ чужие файлы без записи в STATUS.md.
```
